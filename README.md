# 日報アプリ

スマートフォンから手軽に日報を入力・編集し、その場で基幹システム連携用のCSVを出力・保存できるPWAアプリケーションです。

## 開発背景
職場の既存の日報管理システムにおいて、「入力インターフェースの使いにくさ」や「編集機能無し」により、現場作業者の入力負荷が高いことやデータの一貫性がないという課題がありました。
この課題を解決するため、**「現場での入力体験を簡単かつ最速にする」**ことに特化した専用ツールを個人開発しました。

## アプリの目的
- **入力負荷の軽減**: 最小限のタップ数で、正確な製品コードを含んだ日報を作成する。
- **オフライン対応**: 地下や倉庫などの電波環境が不安定な現場でも、ストレスなく動作させる。
- **データ連携の円滑化**: スマートフォンで作成したデータをCSV出力し、そのまま既存の基幹システムへインポート可能にする。

## 特徴・こだわり
### 1. 徹底した現場目線のUX
- スマートフォンでの片手操作を想定したUI設計。
- 曖昧な製品名から基幹システム用コードを自動補完するマスタ連携機能。

### 2. サーバーレス・最小構成
- サーバーやデータベースを介さず、ブラウザのローカルストレージで完結。
- 導入コストとセキュリティリスク（個人情報保持）を最小限に抑制。

### 3. PWA (Progressive Web App)
- サービスワーカーの実装により、完全オフライン環境での起動・動作を実現。
- ホーム画面に追加することで、ネイティブアプリ同様の操作感を提供。

### 4. 実務に即したデータ出力 (CSV)
- **基幹システム互換**: ヘッダーなし、特定の日時セパレータ（スペース2つ）など、既存システムのインポート形式を厳密に再現。
- **Excel対応**: 事務現場での閲覧を想定し、BOM付きUTF-8エンコードを採用。

## 技術スタック
- **Frontend**: React 19, TypeScript, Vite
- **Styling**: Tailwind CSS v4
- **PWA**: Vite PWA Plugin

## セキュリティについて
本リポジトリでは機密保護のため、製品コード等のマスタデータはソースコードから分離しています。
- **GitHub**: ダミーデータによる動作確認用の公開コード。

## データ仕様
出力されるCSVは以下の構成となります：
1. **ID**: 自動連番
2. **日時**: `YYYY/MM/DD  HH:MM:SS` (スペース2つ)
3. **内容**: 作業内容
4. **製品名**: 基幹システム用製品コード
